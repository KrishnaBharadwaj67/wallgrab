WallGrab Development Summary - Optimization Phase
==================================================

Project Overview:
----------------
Optimizing the WallGrab desktop application's scraping engine to reliably resolve high-resolution wallpaper links and bypass bot-detection mechanisms.

What We Did So Far:
------------------
1.  Backend Overhaul (Rust):
    - Rewrote 'scraper.rs' to implement site-specific strategies for 4kwallpapers.com and wall.alphacoders.com.
    - Implemented a two-tiered strategy for 4kwallpapers.com: Listing pages now infer high-res URLs (3840x2160) directly for speed, while detail pages parse all available resolutions.
    - Enhanced AlphaCoders logic to strip thumbnail prefixes and access original source files.
    - Added 'detail_page_url' to the WallpaperInfo struct to enable stateful fallbacks.

2.  Downloader Robustness:
    - Updated 'downloader.rs' with comprehensive browser headers:
        * Standard Chrome User-Agent (Windows 10) to bypass Cloudflare/bot checks.
        * Proper 'Referer' headers to prevent hotlink blocking.
        * Specific 'Accept' headers for image types.
    - Implemented a "404 Fallback" mechanism: If an inferred high-res URL fails, the downloader automatically fetches the wallpaper's detail page, scrapes the "real" resolution links, and retries the download.

3.  IPC Bridge & Frontend:
    - Updated 'main.ts', 'preload.ts', and 'types.ts' to pass 'detail_page_url' from the scraper to the downloader.
    - Fixed a bug in 'App.tsx' where URL query parameters were being stripped, which broke high-quality links for certain providers.

Why We Did It:
-------------
- Quality Assurance: The app was previously downloading low-res thumbnails (e.g., 400x255) instead of available 4K/5K versions.
- Reliability: Site changes and protection mechanisms (like AlphaCoders' Cloudflare) were causing 403/404 errors.
- Performance: Smart URL inference saves bandwidth by avoiding unnecessary detail page loads, but we needed a fallback for when the guess is wrong (e.g., mobile-only wallpapers).

Problems Faced Along The Way:
----------------------------
1.  Bot Detection: Websites like AlphaCoders use Cloudflare which blocks the default 'reqwest' User-Agent. This resulted in immediate 403 Forbidden errors.
2.  Hotlink Protection: Many wallpaper sites check the 'Referer' header. Downloading directly without it often returned 403 errors or placeholder images.
3.  The "4kwallpapers Guessing Game": Guessing the 4K URL (e.g., adding -3840x2160-) works 90% of the time but fails if the image doesn't exist in that exact resolution (like mobile or ultrawide). This required building a recursive scraping fallback.
4.  Query Parameter Stripping: The frontend was over-eagerly cleaning URLs, which removed important quality/formatting parameters used by CDNs like Unsplash.
5.  NAPI-RS Compilation: Structural changes to the Rust types required a full rebuild and redeployment of the '.node' binary to keep the TypeScript definitions in sync.

Status:
-------
The core scraping engine is now significantly more robust, providing true 4K resolution across target sites with reliable fallback and bypass mechanisms.
